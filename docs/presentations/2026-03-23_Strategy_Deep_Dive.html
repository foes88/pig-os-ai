Q1
글로벌 양돈 데이터 표준 API 아키텍처
CORE ANSWER
Schema-per-tenant + TimescaleDB + PowerSync 조합으로
500~5,000두 농장 수천 개를 안정적으로 수용한다.
외부 API 표준은 ICAR ADE(국제 축산 데이터 교환)를 채택한다.

멀티테넌시 — Schema-per-tenant

멀티테넌시란? 하나의 시스템에서 여러 농장을 운영하되, 각 농장의 데이터는 완전히 분리된 공간(스키마)에 저장하는 구조. A농장 데이터가 B농장에 절대 보이지 않는다.
접근법확장 한계테넌트 격리적합성
Database-per-tenant~10개최대과잉
Schema-per-tenant수백~10,000개강력최적
Row-level (RLS)수백만약함시기상조

PgBouncer Transaction Pooling으로 커넥션 충돌 해소. Crunchy Data 사례: DB 비용 62% 절감, 지연시간 81% 개선.

TimescaleDB — IoT 데이터의 정답

PostgreSQL 확장으로 설치. 관계형 번식 기록과 시계열 센서 데이터가 동일 DB에 공존. 시간 기반 집계 쿼리 일반 PostgreSQL 대비 979배 빠름, 압축률 90%+.

결정적 장점: 번식 기록과 환경 데이터를 직접 JOIN — 분만 중 돈사 온도와 자돈 생존율 상관관계 분석 가능.

PowerSync — 오프라인 동기화

CRDT는 구조화된 농장 데이터에 부적합(Cinapse가 CRDT→PowerSync 마이그레이션). PostgreSQL WAL 기반 논리 복제로 네이티브 앱의 로컬 SQLite와 자동 동기화. 충돌 해결은 LWW 기본 + 서버 측 비즈니스 규칙 검증.

성능 목표: API 응답 p95 <200ms(CRUD), <500ms(대시보드), 오프라인 연결 복구 후 <5초, 테넌트 간 데이터 누출 제로.
Q2
SaaS + 데이터 API 결합 수익모델
CORE ANSWER
SaaS 구독(ARR 60~70%) + B2B 데이터 API(15~25%) 이중 수익 구조.

Q3
투자 관점에서 마케팅 리소스를 어디에 집중해야 하는가
CORE ANSWER
전 시장 동시 오픈하되, 같은 돈으로 가장 많은 결과를 내는 시장에 마케팅을 집중한다.
판단 기준: 경쟁 강도 × 확보 비용 × 전환 장벽 × 확산 속도.

투자 효율 판단 기준

핵심 질문: "같은 돈과 시간을 쓸 때, 어디서 더 많은 농장을 확보할 수 있는가?"

판단 기준질문베트남 (집중)미국 (콘텐츠)
경쟁 강도이미 쓰는 SW가 있나?없음 (그린필드)MetaFarms/PigCHAMP 있음
확보 비용농장 1개 확보에 얼마?거의 0 (무료+커뮤니티)콘텐츠+시간 많이 듦
전환 장벽기존 SW에서 갈아타야 하나?갈아탈 것도 없음데이터 마이그레이션 필요
확산 속도얼마나 빨리 퍼지나?빠름 (커뮤니티 밀집)느림 (개별 접촉)
결론: 경쟁자가 없고, 비용이 낮고, 전환 장벽이 없는 시장에 마케팅을 집중하는 것이 투자 효율이 가장 높다. 이것은 "순서"의 문제가 아니라 "같은 돈으로 어디서 더 많은 결과를 내는가"의 문제다.

경쟁사 대비 핵심 차별화
차별화 1
클라우드 네이티브 + 오프라인 퍼스트
경쟁사는 데스크톱→클라우드 전환 중. PigOS AI는 처음부터 클라우드 + PowerSync 완전 오프라인

MetaFarms는 북미, Agriness는 남미에 국한. PigOS AI가 아시아+미국+남미+한국 연결하면 세계 최초

CONCLUSION

ONE LINE
승부는 벤치마크 데이터의 네트워크 효과를 얼마나 빠르게 점화하느냐에 달려 있다.
무료로 농장을 대량 유입 → 50만 모돈 임계점 달성 → 유료 전환 + B2B 데이터 판매 동시 가능.
베트남이 유일한 1순위. 경쟁사 부재 + 산업 기업화 + 디지털 인프라 + 한국 브랜드 신뢰. 연 50~100개 농장 온보딩으로 2~3년 내 동남아 양돈 SaaS 표준이 되는 것이 현실적 목표.
