AGENDA 03
기술 스택 — FastAPI + PostgreSQL
약 3분 · 기술 결정 사항
"백엔드는 FastAPI(Python)로 갑니다.

Java vs Python 고민이 있었는데, 결론은 명확합니다.
AI 분석 기능을 만들려면 Python이 필수입니다.
다른 기술로 시작하면 나중에 Python을 따로 붙여야 해서 이중 작업이 됩니다."
Java 전환은 언제?
지금은 안 합니다. 농장 2,000개, 동시 사용자 5,000명까지 FastAPI로 충분.
전환 트리거: API P95 > 500ms 2주 지속 / 월간 2,000농장 초과 / Enterprise SLA 99.9% 요구 시
"지금 Java로 시작하면 개발 속도가 2~3배 느려집니다.

DB는 PostgreSQL + Schema-per-tenant.
농장별 스키마 분리로 데이터 완전 격리. 규정 준수, 백업도 농장 단위로 가능합니다.
IoT 센서 데이터는 TimescaleDB로 시계열 최적화합니다."
BOTTOM LINE
FastAPI + PostgreSQL (Schema-per-tenant) + TimescaleDB(IoT).
Java 전환은 성능 문제가 예측될 때 시작한다.

약 5분 · 전략 핵심
"마지막으로 가장 중요한 부분입니다. 어디를, 어떻게, 어떤 순서로 공략할 것인가.

"수익 모델은 이중 구조입니다.

1차 — SaaS 구독 (ARR의 60~70%)
모돈당 과금. 500두 이하는 영구 무료. 500두 이상부터 $2~$4/모돈/년.

2차 — B2B 데이터 API (15~25%)
농장 데이터가 모이면 사료회사, 종돈사, 제약사가 돈을 냅니다.
Cargill이 Agriness(브라질, 200만 모돈)에 전략적 투자한 것이 증거입니다.
사료회사 한 곳이 FCR 벤치마크 데이터에 연 $50K~$500K를 지불합니다."
"진입 순서는 베트남이 1순위입니다.

이유 5가지:
1. 양돈 전문 SaaS 경쟁자가 없음 — 그린필드
2. 산업 기업화 가속 — 농장 수 1,000만→200만 통합 중
3. 디지털 인프라 동남아 최고 — 4G 99.8%, 모바일 데이터 월 $8
4. 한국 브랜드 신뢰도 높음
5. 사료회사 채널 풍부 — De Heus, Cargill Vietnam

사료회사 영업사원이 농장 방문 시 같이 소개하는 모델로 CAC를 $300~$800으로 압축합니다.

브라질은 제외합니다. Agriness가 96% 독점. CAC/ARR 비율이 2배 이상이라 효율이 안 나옵니다."

APPENDIX
예상 Q&A 대비
참고용 · 예상 질문 5개
Q1. "다 좋은데 돈은 언제 버나?"
Phase 1 목표는 수익이 아닙니다.
활성 농장 300개 확보가 기준점입니다.
300농장의 30% 전환 시 → 연 $160K~$450K ARR 진입 가능합니다.
Q2. "PigKnows는 이미 글로벌 확장한다는데 이길 수 있나?"
PigKnows가 가지지 못한 것 3가지:
1. 무료 진입 — Valstone 인수 후 구조상 무료 불가
2. 오픈 API — 폐쇄형 수직통합 전략과 모순
3. 소농 최적화 — 대형 인테그레이터 중심, 동남아 소농 미대응
이 3가지 공백에서 우리가 먼저 자리를 잡으면 됩니다.
Q3. "FastAPI로 시작했다가 Java로 바꾸면 처음부터 다시 짜야 하는 거 아닌가?"
비즈니스 로직 재작성은 필요합니다.
그러나 DB 스키마, API 설계, 프론트엔드는 그대로 씁니다.
그 시점이면 수익도 나고 있을 겁니다.
지금 Java로 시작하면 개발 속도가 2~3배 느려집니다.
Q4. "DB를 처음부터 새로 만들면 시간이 더 걸리지 않나?"
오히려 더 빠릅니다.
기존 시스템을 분석하고 맞추느라 드는 시간이 없어집니다.
글로벌 양돈 표준에 맞춰 깨끗하게 설계하면 확장도 쉽습니다.
