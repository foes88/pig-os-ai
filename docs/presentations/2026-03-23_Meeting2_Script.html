AGENDA 01
나라마다 원하는 게 다르다
약 5분 · 5개 권역 분석 결과
"저희가 타겟으로 잡은 5개 권역 — EU, 미국, 브라질, 동남아, 중국.
이 나라들의 양돈 현실이 완전히 다릅니다.

예를 들면, 덴마크는 PSY가 35.6입니다. 세계 최고.
반면 동남아는 10~26까지 편차가 큽니다. 베트남 소농은 인터넷도 없습니다.

미국은 PRRS 피해가 연간 $6.6억 수준이고, PQA Plus 인증 없으면 유통사 계약이 안 됩니다.
EU는 항생제 사용 보고가 법적 의무입니다.
동남아는 ASF(아프리카돼지열병)가 상재화되어 있어서 질병 경보가 생존 문제입니다.

— 잠깐 멈춤 — 그래서 하나의 제품으로 6개 나라를 다 커버하려면, 설계 초기부터 이걸 반영해야 합니다.

다행히 공통 분모가 있습니다.
어느 나라든 모돈 관리의 기본은 같습니다 — 교배, 분만, 이유.
여기에 오프라인 모바일과 다국어를 붙이면, Phase 1에서 어디든 쓸 수 있습니다.

지역 특화 기능 — 미국의 Feed Mill 연동, EU의 항생제 보고, 동남아의 ASF 경보 —
이건 Phase 2부터 모듈로 분리 개발합니다."
KEY POINT
Phase 1에서 3개 지역 공통 기능(교배/분만/이유 + 오프라인 + 다국어)만 만들면 어디든 쓸 수 있다. 지역 특화는 Phase 2부터.
"현지화 명세도 다 정리했습니다.
무게 단위(kg vs lbs), 통화(원/달러/동), 날짜 포맷, 심지어 알림 채널까지 —
베트남은 Zalo, 브라질은 WhatsApp.
이걸 DB의 country_configs 테이블 하나로 다 관리합니다."
AGENDA 02
경쟁사 분석 — 5개 공백을 찾았다
약 5분 · 9개사 딥다이브 결과
"9개 경쟁사를 전부 뜯어봤습니다.
PigCHAMP, PigKnows, MetaFarms, Cloudfarms, MTech, Porcitec, Agriness, LeeO, S&W.

먼저 가격입니다.
거의 전부 비공개입니다. 영업팀한테 연락해야 견적이 나옵니다.
확인된 건 MetaFarms가 출하돈당 $0.05~$0.25, Cloudfarms가 $500/년부터.
MTech은 대형 인테그레이터 전용이라 아예 가격 자체가 없습니다.

가격을 숨기는 게 업계 관행입니다. 우리는 이걸 깨겠다는 겁니다.
"그리고 5개 시장 공백을 찾았습니다.
이건 어떤 단일 경쟁사도 동시에 충족 못하는 겁니다."
WHITE SPACE 1
아시아 특화 클라우드 SaaS — Cloudfarms가 24개 언어 지원하는데, 아시아어는 없습니다
WHITE SPACE 2
ESG + 양돈 운영 통합 — 탄소/동물복지 자동 산출 솔루션이 아예 없습니다
WHITE SPACE 3
개체 IoT + 모돈 관리 통합 — LeeO는 IoT만, PigCHAMP은 모돈만. 합친 건 없습니다
WHITE SPACE 4
중소 농장 합리적 가격 — 대형 전용이거나 가격 비공개. 소농은 접근 불가
WHITE SPACE 5
AI 분석 민주화 — MTech/LeeO AI 대형 전용. 우리는 SaaS에 AI 포함
"Phase 1 출시 즉시 5개가 차별화됩니다.
아시아어, 무료, 단위 자동 전환, 아시아 규정 대응, 가격 투명 공개.
이건 구조적 공백이라서 경쟁사가 단기간에 따라오기 어렵습니다.

— 대표님 질문 예상 포인트 — '무료로 주면 돈은 언제 버냐?'

Phase 1 목표는 수익이 아닙니다. 활성 농장 300개 확보가 기준점입니다.
300농장의 30%만 유료 전환해도 연 $160K~$450K ARR 진입이 가능합니다.

피그플랜 27년 개발 비용은 이미 회수됐고,
글로벌 데이터 자산 — 저희가 Pigsignal이라 부르는 — 확보가 진짜 목적입니다."
AGENDA 03
기술 스택 — FastAPI + PostgreSQL
약 3분 · 기술 결정 사항
"백엔드는 FastAPI(Python)로 갑니다.

Java vs Python 고민이 있었는데, 결론은 명확합니다.
AI/ML 분석 — 질병 경보, 최적 출하 예측 — 이게 Phase 2~3에 핵심인데,
Python 생태계 없이는 별도 ML 서버 구축 비용이 2배입니다.
그리고 피그플랜 기존 분석 로직을 바로 재사용할 수 있습니다."
Java 전환은 언제?
지금은 안 합니다. 농장 2,000개, 동시 사용자 5,000명까지 FastAPI로 충분.
전환 트리거: API P95 > 500ms 2주 지속 / 월간 2,000농장 초과 / Enterprise SLA 99.9% 요구 시
"지금 Java로 시작하면 개발 속도가 2~3배 느려집니다.

DB는 PostgreSQL + Schema-per-tenant.
농장별 스키마 분리로 데이터 완전 격리. 규정 준수, 백업도 농장 단위로 가능합니다.
IoT 센서 데이터는 TimescaleDB로 시계열 최적화합니다."
BOTTOM LINE
FastAPI + PostgreSQL (Schema-per-tenant) + TimescaleDB(IoT).
Java 전환은 성능 문제가 예측될 때 시작한다.
AGENDA 04
DB 설계 방향 — 완전 신규
약 2분 · 설계 원칙
"DB 설계 방향을 말씀드리겠습니다.

기존 시스템과 완전히 독립된 신규 설계로 갑니다.

기존 시스템을 분석하고 맞추는 데 시간 쓰는 것보다,
글로벌 양돈 표준에 맞춰 처음부터 깨끗하게 설계하는 게 더 빠르고 확실합니다.

핵심 원칙은 3가지입니다."
원칙 1
Schema-per-tenant — 농장별 데이터 완전 격리. 보안과 규정 준수에 유리.
원칙 2
모듈형 구조 — 11개 모듈 독립 배포. Phase별로 기능을 하나씩 붙여나갈 수 있다.
원칙 3
5개 권역 동시 대응 — 하나의 스키마로 EU/US/BR/SEA/CN 전부 커버. country_configs로 지역별 설정 관리.
"스키마 설계는 이미 49개 테이블, 11개 모듈로 완성된 상태입니다.
기존 시스템 분석 단계를 건너뛰고 바로 개발에 들어갈 수 있습니다."

APPENDIX
예상 Q&A 대비
참고용 · 대표님 예상 질문 5개
Q1. "다 좋은데 돈은 언제 버나?"
Phase 1 목표는 수익이 아닙니다.
활성 농장 300개 확보가 기준점입니다.
300농장의 30% 전환 시 → 연 $160K~$450K ARR 진입 가능합니다.
Q2. "PigKnows는 이미 글로벌 확장한다는데 이길 수 있나?"
PigKnows가 가지지 못한 것 3가지:
1. 아시아어 (한/베/태) — 단기간 추가 불가
2. 무료 진입 — Valstone 구조상 불가
3. 오픈 API — 폐쇄형 수직통합 전략과 모순
이 3가지 공백에서 우리가 먼저 자리를 잡으면 됩니다.
Q3. "FastAPI로 시작했다가 Java로 바꾸면 처음부터 다시 짜야 하는 거 아닌가?"
비즈니스 로직 재작성은 필요합니다.
그러나 DB 스키마, API 설계, 프론트엔드는 그대로 씁니다.
그 시점이면 수익도 나고 있을 겁니다.
지금 Java로 시작하면 개발 속도가 2~3배 느려집니다.
Q4. "DB를 처음부터 새로 만들면 시간이 더 걸리지 않나?"
오히려 더 빠릅니다.
기존 시스템을 분석하고 맞추느라 드는 시간이 없어집니다.
글로벌 양돈 표준에 맞춰 깨끗하게 설계하면 확장도 쉽습니다.
스키마 설계는 이미 49개 테이블, 11개 모듈로 완성된 상태입니다.
Q5. "베트남 농장주가 앱을 실제로 쓸 수 있을까?"
이게 MVP에서 검증해야 할 핵심 질문입니다.
피그플랜 기준, 현장 작업자는 3개 화면만 씁니다:
교배 기록 / 분만 기록 / 이유 기록
이 3개 화면을 최대한 단순하게 만드는 것이 MVP UX 목표입니다.
