# PigOS AI 사업·개발 리스크 평가서

> 작성일: 2026-04-05
> 목적: 낙관론에 치우친 기획에 대한 균형추. 의사결정 전 반드시 검토.
> 톤: 의도적으로 비관적. 좋은 점은 기획서에 이미 많음.

---

## 0. 한 줄 요약

**제품은 만들 수 있다. 그러나 팔 길이 보이지 않는다.**
현재 상태에서 그대로 개발에 착수하면, 2년 뒤 "기능은 많은데 고객이 없는" 제품이 나올 확률이 높음.
가장 큰 리스크는 기술·제품이 아니라 **GTM(시장 진입)**.

---

## 1. 전략 리스크

### 1.1 "무료 → 데이터 수집 → 수익화" 모델의 함정
- 양돈 농가는 B2B SaaS 고객 중 **가장 보수적인 층**. 무료여도 설치/입력하지 않음.
- "무료 뿌리고 나중에 수익화"는 B2C 논리. B2B 농업에서는 거의 작동한 전례 없음.
- 데이터가 임계량에 도달하기 전에 운영비·조직 피로가 먼저 소진됨.

### 1.2 5개 시장(US/CN/SEA/LatAm/KR) 동시 타겟
- 각 시장의 규제·언어·질병체계·유통구조·결제가 모두 다름.
- `country_configs` 같은 기술적 추상화로 해결되는 문제가 아님. **GTM 문제**는 코드로 못 품.
- 초기 조직이 5개 시장을 동시에 감당할 영업·CS·현지화 리소스 없음.
- **현실적 상한: 1개 시장 + 1개 세그먼트.**

### 1.3 글로벌 양돈 시장의 구조적 진입 장벽
| 시장 | 장벽 |
|------|------|
| 미국 | Smithfield/Tyson/JBS 등 수직계열 통합업체가 70%+ 점유. PigCHAMP/MetaFarms와 수십년 관계. 신규 벤더 진입 거의 불가. |
| 유럽 | AgroVision/Farmsoft 장악. GDPR, 각국 규제. |
| 중국 | Muyuan/Wens 자체 개발. 해외 SaaS는 데이터 국외 반출 규제로 사실상 차단. |
| 브라질 | JBS/BRF 지배. 포르투갈어. 현지 파트너 필수. |
| 동남아 | CP Group(태국) 지역 장악. 소농 많으나 ARPU 매우 낮음. |

**진입 가능한 틈새**: "중형 독립 농가" 층 — 각국에 흩어져 있고 디지털화가 가장 느린 층. CAC 높고 LTV 낮음.

### 1.4 "피그플랜과 별도" 전략의 이중 리스크
- 기존 브랜드/네트워크 미사용 시 해외 농가 입장에서 **그냥 신생 한국 스타트업**.
- 27년 노하우가 "머릿속"에만 있으면 해자가 아님. 제품/데이터/네트워크에 박혀야 해자.
- **전제 조건**: 피그플랜 데이터 자산(질병코드, 백신, 벤치마크, 이벤트 템플릿, 농가 KPI 분포)이 PigOS AI로 이관 가능해야 이 전략이 성립.
  - 현재 확인됨: 이관 가능. → 이 리스크는 부분 해소. 단, **전략 메시지에 제대로 반영되어 있지 않음**.

### 1.5 "캐시카우로 메꾸는" 구조의 축복과 저주
**축복**
- 런웨이 걱정 없음. VC 의존 시 시리즈 A 받기 전에 죽을 사업을 버틸 수 있음.

**저주 (이쪽이 더 위험)**
- 돈 걱정이 없어서 **시장 검증 단계를 건너뜀**.
- 내부 임원/주주 의견 반영하다 제품이 누더기가 됨.
- 스코프 축소 압력이 없어 11 모듈·5개 시장 전부 하려는 유혹.
- 3년 뒤 "성과 없으니 접자" 결정이 나고, 그때쯤 경쟁사는 이미 자리 잡음.

> 모회사 캐시카우가 있는 신사업의 가장 흔한 실패 패턴.

## 2. GTM(시장 진입) 리스크 — **최대 리스크**

### 2.1 채널 파트너 부재
- 현재 상태: **현지 파트너 후보 0건**.
- B2B 농업 SaaS는 본질적으로 **채널 게임**: 사료회사, 동물약품사, 수의사 협회, 협동조합, 정부 농업부가 농가로 가는 유일한 길.
- 이 길을 뚫지 못하면 5년 써도 10 농가 못 모음.

### 2.2 마케팅으로는 풀리지 않는 문제
- 양돈 농가는 구글 검색으로 SaaS 안 삼.
- 박람회 부스·SNS 광고·콘텐츠 마케팅 모두 전환율 거의 0.
- "홍보 마케팅이 어렵다"는 인식은 정확하지만, **홍보 문제가 아니라 채널 문제**.

### 2.3 유닛 이코노믹스 불일치
| 항목 | 수치(추정) |
|------|-----------|
| 농가당 ARPU | $500–2,000/년 |
| 양돈 박람회 부스 1회 | $30,000–50,000 |
| 현지 영업 인력 1명 | $100,000+/년 |
| 직접 세일즈 CAC | 농가당 수천 달러 |

→ **현지 파트너/리셀러 없이는 구조적 적자.** 이 방정식은 제품 품질로 풀리지 않음.

### 2.4 "첫 고객 10곳"의 부재
- 현재 확보 가능한 글로벌 첫 고객 10곳의 구체적 경로 없음.
- 이 질문에 답하지 못한 상태에서 개발 착수는 매우 위험.

## 3. 기술·개발 리스크

### 3.1 MVP 스코프 과대
- 현재 기획: **11 모듈, 49 테이블, 5개 시장, FastAPI→Spring Boot 전환, K8s, TimescaleDB, WatermelonDB, Schema-per-tenant**.
- 이는 **시리즈 B 단계 회사의 스택**. MVP에 적용 시 출시 전 조직 번아웃.
- 실제 사용 패턴이 나오기 전에 스키마를 굳히면, 1년 뒤 대부분 갈아엎어야 함.

### 3.2 스키마가 한국 워크플로우에 맞춰질 위험
- 글로벌 농가 워크플로우는 한국과 근본적으로 다름.
  - 미국: 대형 수직계열 통합업체 중심, 다사이트 생산 시스템, 계약 사육
  - 유럽: 협동조합, 동물복지·GDPR 규제 필드
  - 동남아: 소농, 저품질 네트워크, 공용 스마트폰 입력
- "한국 스키마 + country_configs"로는 이 차이를 다 못 담음. 필드가 아니라 **모델 자체가 다름**.

### 3.3 레거시 탈출 자체는 타당, 단 재설계가 "또 다른 레거시"가 될 위험
- DB 구조 새로 짜는 것 자체는 **100% 타당** (레거시로 글로벌 불가).
- 그러나 실사용자 없이 설계된 새 스키마는 **깨끗한 새 레거시**가 됨.
- 실제 고객 피드백 루프 없이 설계된 49 테이블은 2년 뒤 재설계 대상.

### 3.4 이중 제품 운영 부담 (향후)
- 한국=피그플랜 / 해외=PigOS AI 분리 운영 시, 양쪽 코드베이스·데이터·팀을 따로 유지해야 함.
- 27년 자산을 PigOS AI로 가져오더라도, **동기화·이중 유지보수 비용**이 지속적으로 발생.
- 장기적으로 "언젠가 합쳐야 한다"는 부담이 계속 따라옴.

### 3.5 오프라인 동기화의 난이도
- WatermelonDB + Last-Write-Wins 선택은 합리적이나, 실제 구현은 **엣지케이스 지옥**.
- 월마감 잠금(`period_locks`)과 오프라인 동기화의 충돌 시나리오는 사전에 명세 필요.
- 이 한 영역만으로도 개발자 1명의 6개월짜리 작업.

### 3.6 멀티테넌시 선택의 무게
- Schema-per-tenant는 격리성은 좋으나 **마이그레이션·백업·모니터링 비용이 선형 증가**.
- 고객 100곳 미만에서는 단일 스키마 + `tenant_id`가 10배 쉬움.
- 현재 스케일(고객 0명)에 비해 과대 설계.

## 4. 조직·실행 리스크

### 4.1 BD/영업 인력 부재
- 현재 조직에 **"양돈 업계를 알면서 글로벌 BD를 할 수 있는 인력"** 유무가 불투명.
- 이 사업에서 이 인력은 CTO보다 10배 중요.
- 없으면 제품이 아무리 좋아도 팔리지 않음.

### 4.2 피드백 루프 단절
- 글로벌 고객 없이 한국에서 설계 → 출시 → 해외 테스트는 피드백 사이클이 6~12개월.
- 한국 고객 먼저 쓰는 경로가 없으면 초기 품질이 낮을 수밖에 없음.

### 4.3 "피그플랜과 별도" 결정의 숨은 원인
- 기술적 이유(레거시 탈출)만이면 합리적.
- 그러나 조직 내부 정치/주주 구조/권한 분리가 숨은 원인이라면, **새 코드베이스로 우회할 수 없는 문제**.
- 이 동기가 명확해야 3년 뒤 통합·분리 결정에서 흔들리지 않음.

## 5. 시장·외부 리스크

### 5.1 경쟁사 선점
- PigCHAMP(美), AgroVision(유럽)은 이미 수십 년 레퍼런스 보유.
- 동남아 신흥 플레이어: Farmcloud(태국), HerdDogg, Vetafarm 등이 저가 모바일 퍼스트로 진입 중.
- "늦지 않았다"고 보기 어려움. 시간 싸움.

### 5.2 규제 리스크
- GDPR(EU), PIPL(중국), LGPD(브라질), 농업 데이터 주권법(각국 신설 중).
- 농장 데이터는 "개인정보는 아닌데 준개인정보"로 취급되는 추세.
- 5개 시장 전개 시 법무 비용만 연 수억.

### 5.3 ASF/신종 질병 충격
- ASF 재확산 시 농가 투자 여력 급감 → SaaS 해지율 급증.
- 반대로 질병 대응 수요로 기회가 될 수도 있으나, 단기 매출은 타격.

### 5.4 환율·결제
- 5개 시장 = 5개 통화 + 현지 결제 수단 + 송금 규제.
- 동남아는 신용카드 결제율 낮음. 현지 결제 파트너 필수.

## 6. 재무 리스크

### 6.1 캐시카우 의존의 한계
- "피그플랜으로 메꾼다"는 피그플랜이 **앞으로 5년간 흑자 유지**를 전제.
- 국내 양돈 시장 축소, 농가 고령화, 경쟁사 출현 등으로 피그플랜 자체가 흔들리면 PigOS AI도 동반 위험.

### 6.2 BEP 도달 시점 불투명
- 글로벌 B2B SaaS는 통상 **5~7년**에 BEP.
- 그 동안 누적 적자를 감당할 의지/자금/거버넌스가 필요.
- 3년 시점 중간 평가에서 "접자" 결정이 나올 가능성을 사전에 인지해야 함.

### 6.3 데이터 수익화의 현실성
- "사료회사/제약사에 데이터 판매"는 매력적이나, **임계량(데이터 커버리지)에 도달해야 가능**.
- 동남아 중형 농가 기준 최소 500~1000 농가 × 2년 실사용 데이터 필요.
- 즉 **빨라야 4~5년차 매출**. 그 전까지는 SaaS 구독료만으로 버텨야 함.

## 8. 가장 중요한 판단 질문 (의사결정 체크리스트)

개발 착수 전 아래 질문에 **명확한 답**이 있어야 함. 흐릿하면 착수 금지.

- [ ] **첫 고객 10곳**을 어떻게 확보할 것인가? (이름 단위로 답 가능한가?)
- [ ] **현지 파트너 후보**가 최소 1곳 있는가? 없다면 언제까지 확보할 것인가?
- [ ] **1개 시장 + 1개 세그먼트**로 줄이라면 어디인가?
- [ ] **BD/영업 인력**이 조직 내 있는가? 없다면 채용 계획은?
- [ ] **3년차까지 매출 0**이어도 의사결정권자들이 버틸 수 있는가?
- [ ] 피그플랜 **데이터 자산 이관**이 전략 메시지의 1순위로 잡혀 있는가?
- [ ] "피그플랜과 별도" 결정의 **진짜 이유**가 기술인가 조직인가? 조직이면 그것부터 풀었는가?
- [ ] MVP 스코프를 **3개 모듈 이하**로 줄일 의지가 있는가?

## 9. 권장 대응 (Do / Don't)

### Do
- **개발 착수 전 6개월간 BD only 모드**. 한국 사료회사(CJ/하림/팜스코/선진) 해외사업부 3~5곳 미팅.
- **MOU 1장 확보 후 개발 시작**. 돈 안 받아도 됨. "파일럿 100 농가"면 충분.
- **MVP는 3개 모듈**: 모돈 카드 + 이벤트 로그 + KPI 계산. 끝.
- **1개 시장**: 베트남 또는 필리핀. 사료회사 이미 진출한 곳.
- **기술 스택은 린하게**: FastAPI + Postgres + Next.js + 모바일 앱 1개. K8s/Timescale/Spring 전환은 고객 100 넘으면 그때.
- **피그플랜 데이터 자산**을 전략 메시지 1순위로.
- **BD 인력 채용**: CTO보다 우선순위 높게.

### Don't
- **무료로 농가 직접 영업** 시도 금지. CAC 0이 아니라 무한대.
- **5개 시장 동시 전개** 금지. 내부 임원 요구가 있어도 버틸 것.
- **11 모듈 49 테이블** 사전 설계 금지. 고객 워크플로우 나오면 점진 확장.
- **Schema-per-tenant / K8s / Spring Boot 전환** 초기 도입 금지.
- **한국 스키마 기반으로 country_configs로 글로벌 커버** 접근 금지.
- "일단 만들고 마케팅은 나중에" 금지.
- 캐시카우 있다고 **시장 검증 생략** 금지.

## 10. 결론

이 사업은 **가능성 있음**. 단, 다음 조건이 동시에 충족될 때만:

1. 피그플랜 데이터 자산을 전략의 1순위 메시지로 쓴다
2. 농가 직접 영업을 버리고 현지 채널 파트너를 확보한다
3. 1개 시장 + 1개 세그먼트로 스코프를 줄인다
4. MVP 기술 스택을 린하게 간다
5. 캐시카우의 함정(느슨한 검증·내부 의견 과다 반영)을 경계한다

**5개 조건 중 1~2개만 충족 → 5년 뒤 조용히 종료.**
**4~5개 모두 충족 → 아시아 PigCHAMP 가능. ARR $50–100M 사이즈.**
