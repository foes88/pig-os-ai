## 섹션 1 — 서비스 개요

> **01 · 서비스 개요**
> PigOS AI가 무엇이고, 왜 지금 만들어야 하는가.

### 1.1 제품 정의

**PigOS AI**는 해외 양돈 농장을 위한 **글로벌 Farm Management SaaS**다. 피그플랜(국내)의 27년 양돈 운영 노하우를 기반으로, 해외 농장주·현장 작업자·수의사에게 번식 관리, 사료 관리, 출하 관리, KPI 대시보드, 오프라인 모바일을 단일 플랫폼에서 제공한다.

| 항목 | 내용 |
|------|------|
| 제품명 | PigOS AI — Global Swine Farm Management SaaS |
| 포지셔닝 | 해외 양돈 농장주·현장 작업자·수의사를 위한 통합 농장 관리 플랫폼 |
| 타겟 시장 | 미국/캐나다, 유럽(EU), 동남아시아(베트남/태국/필리핀) |
| 핵심 기능 | 번식 사이클 관리, 사료·공급망, 출하 관리, KPI 대시보드, 오프라인 모바일 |
| 데이터 역할 | 농장 데이터를 **수신**하는 허브 (IoT·ESF·ERP 연동) — 데이터 판매 사업이 아님 |
| 기술 스택 | FastAPI(Phase 1) → Spring Boot(Phase 2) / React+TS / React Native / PostgreSQL |
| 개발 전략 | Oracle 기존 시스템 분석 → PostgreSQL 재설계 / Schema-per-tenant |

### 1.1-B 피그플랜 자산 — PigOS AI의 출발점

PigOS AI는 빈손으로 시작하는 신규 서비스가 아니다. 국내 양돈 관리 플랫폼 **피그플랜(PigPlan)**의 27년 운영 노하우와 데이터 자산을 직접 계승한다.

| 자산 | 수치 | 의미 |
|------|------|------|
| 누적 데이터 | **1.52억 건** | 한국 양돈 산업 전체의 생산 데이터 |
| 실시간 관리 | **전국 모돈의 32%** | 현재 가동 중인 피그플랜 점유율 |
| 운영 기간 | **1999년~현재 (27년)** | 국내 최장수 양돈 SaaS |
| 성과 기여 | PSY 14두(2010) → 22두(2023) | 한국 양돈 생산성 향상의 데이터 파트너 |

> **이 자산이 핵심이다.** 한국에서 검증된 27년 데이터 패턴과 운영 경험이 글로벌 버전 CORE의 설계 기반이 된다. 경쟁사들이 10년 만에 만든 기능을 CORE는 출시 첫날부터 제공할 수 있는 이유다.

**[ALERT-BLUE]** PSY(Pigs per Sow per Year)는 양돈 농장의 핵심 생산성 지표다. 피그플랜을 쓰는 농장의 평균 PSY가 전국 평균 대비 지속적으로 높다는 데이터가 CORE의 가장 강력한 세일즈 레퍼런스가 된다.

> **참고:** Pigsignal(B2B 데이터 판매)은 별도 사업이다. CORE가 축적한 글로벌 데이터는 익명화·집계 후 Pigsignal에 공급되지만, 이 기획서는 CORE 자체의 Farm Management 전략만 다룬다.

### 2.3 양돈 핵심 지표 — 용어 인포그래픽 [SVG/인포그래픽 컴포넌트]

> **HTML 렌더링 가이드:** `kpi-explainer` 컴포넌트. 3열 카드 그리드. 각 카드: 지표명(JetBrains Mono) + 정의 + 벤치마크 바 차트 + 의미 한 줄.

| 지표 | 정의 | 세계 최고 | 한국 평균 | 왜 중요한가 |
|------|------|-----------|-----------|-------------|
| **PSY** (Pigs/Sow/Year) | 모돈 1두가 1년간 생산하는 자돈 수 | 덴마크 37두 | 22두 | 격차 = 데이터 기반 관리의 ROI |
| **NPD** (Non-Productive Days) | 모돈이 임신/수유 외 상태로 보내는 날 수 | 목표 30일 이하 | ~45일 | NPD 1일 감소 = 두당 수천원 절감 |
| **FCR** (Feed Conversion Ratio) | 체중 1kg 증가에 필요한 사료량 | 목표 2.5 이하 | ~2.8 | 사료비 = 생산비의 70%. FCR 0.1 개선 = 두당 수천원 |
| **MSY** (Market pigs/Sow/Year) | 모돈 1두가 출하까지 생산하는 상품돈 수 | 덴마크 34두 | 19두 | PSY × 생존율. 농장 수익성의 최종 지표 |
| **분만율** (Farrowing Rate) | 교배 후 실제 분만으로 이어진 비율 | 목표 90% 이상 | ~85% | 1% 향상 = 농장 전체 수익 직결 |

**[ALERT-BLUE]** 이 5개 지표가 PigOS AI가 매일 추적하는 핵심 KPI다. 소프트웨어의 가치는 이 수치들을 실시간으로 보여주고, 벤치마크와 비교하고, 개선 방향을 제시하는 것이다.

### 3B.5 시장 공백 지도 (White Space Map)

> **[WHITESPACE-INTRO]**
> WHITE SPACE MAP
> 전 경쟁사가 놓친 5개 공백 — PigOS AI의 진입 지점
> 13개사 분석 결과. 어떤 단일 솔루션도 아래 5가지를 동시에 충족하지 못한다.

#### 공백 1: 아시아 특화 클라우드 네이티브 SaaS

- **현황:** Cloudfarms 24개 언어 지원하지만 한국어/베트남어/아시아 규정 없음. MetaFarms/PigKnows는 북미 전용. AgroVision은 유럽 중심.

#### 공백 2: ESG 보고 + 양돈 운영 데이터 통합

- **현황:** EU 규정 대응(AgroVision HIT/TAM, Porcitec IoT)은 존재하나 탄소 배출/동물복지 점수/항생제를 운영 데이터에서 자동 산출하는 솔루션 없음.
- **기회:** 이마트/코스트코/월마트 바이어의 ESG 요구에 즉시 대응

#### 공백 3: 개체 IoT 추적 + 완전한 모돈 관리 통합

- **현황:** LeeO(개체 IoT 최강) vs PigCHAMP(모돈 관리 최강) — 둘 다 통합 못함.
- **기회:** 모돈 관리 전 기능 + 개체 수준 IoT + AI 이상 탐지 — 단일 플랫폼

#### 공백 4: 중소 농장을 위한 합리적 가격의 완전 기능 SaaS

- **현황:** MTech(대형 전용), PigCHAMP/Porcitec(가격 비공개), Cloudfarms($500+/년). 100~1,000두 농가가 합리적 가격으로 쓸 선택지 부재.
- **기회:** 무료 진입 → 투명한 Per Sow 가격 — 전 경쟁사 미실현

#### 공백 5: AI 분석의 민주화 (대형 인테그레이터 전용 → 중소 농장) ★FEATURED

- **현황:** MTech ML 예측/LeeO 성장 알고리즘 — 모두 대형 고객 전용. 중소 농장이 접근 가능한 AI 예측(질병 경보/최적 출하/사료 효율)은 전무.
- **기회:** SaaS 기본 가격에 AI 분석 포함 — 중소 농장 시장에서 압도적 가치 제안

**[BOTTOM LINE]** 전 경쟁사가 놓친 5개 공백이 PigOS AI의 진입 지점이다.

## 섹션 5 — 가격 모델 패턴 분석

> **05 · 가격 모델 패턴 분석**
> 양돈 SW 시장에서 검증된 과금 모델 5가지 비교.

| 모델 유형 | 특징 | 한계 | 대표 사례 |
|-----------|------|------|-----------|
| `SaaS / Per Sow` | 모돈 수 기반 월/연 구독. 규모 비례. | 소형 농장 ARPU 낮을 수 있음 | Cloudfarms ($500+/yr) |
| `SaaS / Per User` | 사용자 계정 수 기반 과금 | 현장 작업자 많아 불만 가능 | 양돈 전문에선 비주류 |
| `Enterprise Quote` | 대형 Integrator 연간 계약 | 영업 사이클 길고 복잡 | PigKnows, MetaFarms 추정 |
| `HW 결합형` | IoT·ESF와 SW 번들. Lock-in. | 초기 비용 높음 | MetaFarms+Jyga GESTAL |
| `무료 제공 후 유료 전환` | 기본 무료 → 임계점 후 과금 | 전환율 관리 어려움 | 양돈 전문은 드물지만 SEA 진입 시 유효 |

### 양돈 SW 시장의 가격 투명성 문제

양돈 관리 SW 업체의 **대다수가 가격을 공개하지 않는다**.

- 모돈 수, 사이트 수, 필요 모듈이 천차만별 → 단일 가격 제시 어려움
- 가격 비교를 어렵게 만들어 영업 협상력 확보
- Enterprise 고객은 관계 기반 계약 선호

**[ALERT-BLUE]** PigOS AI 기회: 유료 전환 시 진입 Tier의 **가격 투명성을 차별화 포인트**로 활용. 기본 구독 가격 공개는 신뢰 제고와 Self-serve 전환을 동시에 달성.

**[BOTTOM LINE]** Per Sow 구독 + 오픈 연동 + 무료 진입이 시장을 여는 조합이다.

## 섹션 7 — 프라이싱 전략

> **07 · 프라이싱 전략**
> 완전 무료 출시 → 임계 농가 수 도달 후 유료화 전환.

**[ALERT-GREEN]** 핵심 방향: 완전 무료 출시 → 임계 농가 수 도달 후 유료화 전환. 초기에는 모든 기능을 무료로 제공. 시장 침투율과 데이터 확보가 최우선. "첫 번째 글로벌 디지털 양돈 관리 표준을 만드는 것"이 목표다.

- **Phase 2 — 유료 전환 시 Per Sow 기반 과금.** 농장 규모에 비례.
- **지역별 가격 차등:** 동남아 LOW / 미국·유럽 MID~HIGH. PPP 기반 현지화.
- **Module Add-on:** CORE 구독 + 규정·Feed·Genetics·IoT 별도 Add-on.

  (Per Sow 구독)       (바이럴 확산)
       │                    │
       ▲                    ▼
  4. 임계점 도달  ←──  3. 데이터·레퍼런스
  (유료 전환 트리거)    (Pigsignal 자산화-별도)

## 섹션 8 — 리스크 분석

> **08 · 리스크 분석**
> 잘못될 수 있는 것들. 대응 방안과 함께.

**[ALERT-RED]** 리스크를 모르는 것이 가장 큰 리스크다. 아래 5가지는 PigOS AI 실행 과정에서 가장 현실적으로 발생할 수 있는 위협이다.

### 8.1 리스크 매트릭스

> **HTML 렌더링 가이드:** `risk-matrix` 컴포넌트. 각 리스크 카드: 빨간 왼쪽 보더 + 발생 확률(HIGH/MED/LOW) 배지 + 영향도 배지 + 현황 + 대응 방안.

#### 리스크 1: Oracle→PostgreSQL 마이그레이션 실패
`발생확률: MEDIUM` `영향도: HIGH`

- **현황:** 27년 Oracle DB에 누적된 스키마 복잡도 미파악. 레거시 의존성 존재 가능.
- **징후:** Oracle MCP 분석 시 예상보다 많은 커스텀 프로시저·트리거 발견.
- **대응:** Oracle MCP 분석 2주 집중 선행 → 마이그레이션 범위 확정 후 개발 착수. 해외용 CORE DB는 국내 레거시와 완전 분리 설계로 의존도 최소화.

#### 리스크 2: 무료 기간 장기화 → 유료 전환 심리적 저항
`발생확률: HIGH` `영향도: HIGH`

- **현황:** 무료 제공 기간이 길수록 "당연히 무료"로 인식. 유료 전환 시 이탈 급증 가능.
- **징후:** Phase 1 사용자가 "왜 갑자기 돈 받아요?" 반응 집중 시.
- **대응:** 출시 첫날부터 "Phase 1 무료 — 임계점 후 유료 전환 예정" 명시. Early Adopter 혜택(영구 50% 할인 등) 사전 예고. Grandfather 정책 설계 선행.

#### 리스크 3: Valstone이 SEA/EU 시장에 먼저 진입
`발생확률: MEDIUM` `영향도: HIGH`

- **현황:** PigKnows GM "글로벌 확장" 선언. Valstone 자금력 충분.
- **징후:** PigKnows 스페인어/포르투갈어 외 언어 추가, 동남아 영업팀 채용 공고.
- **대응:** SEA Phase 1 진입을 6개월 내로 단축. 현지 사료회사 파트너십 선점이 가장 강력한 방어막. "아시아 특화"는 Valstone이 단기간에 복제 불가.

#### 리스크 4: 오프라인 동기화 충돌(Conflict Resolution) 실패
`발생확률: MEDIUM` `영향도: MEDIUM`

- **현황:** 동일 모돈을 2개 기기에서 오프라인 동시 편집 후 동기화 시 데이터 충돌.
- **징후:** 파일럿 농장에서 "분만 기록이 사라졌다" 민원.
- **대응:** WatermelonDB vs SQLite + CRDT 기반 충돌 해결 기술 2개월 내 POC. 최악의 경우 "마지막 편집자 우선" + 충돌 이력 보존으로 단순화.

#### 리스크 5: 아시아 개인정보보호법 위반
`발생확률: LOW` `영향도: CRITICAL`

- **현황:** 베트남 PDPD(2023), 태국 PDPA(2022), 한국 PIPA 각각 상이한 요건.
- **징후:** 농장주 데이터를 국내 서버에 저장 시 현지 데이터 역외이전 규정 위반.
- **대응:** SEA 진입 전 각국 데이터 현지화 요건 법무 검토 필수. 초기에는 AWS/GCP 현지 리전(싱가포르) 사용으로 리스크 최소화.

**[BOTTOM LINE]** 리스크 2(유료 전환 저항)와 리스크 3(Valstone 선점)이 최우선 관리 대상이다.
